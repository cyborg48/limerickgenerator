About

Developed by a bored sixteen year old who loves programming as much as she does poetry, this site retrieves from the three APis listed above. It was programmed using HTML, CSS, and JavaScript, using Ajax to retrieve the data from the databases. Based on a few randomly generated conditions (i.e., subject's name and occupation), the program searches for words that rhyme or relate with these parameters using Datamuse and then checks if the word is the correct part of speech using Merriam Webster's Dictionary API. If the word is the incorrect part of speech, the program retrieves a new word and starts again, looping until it finds the correct word. Finally, the Linguatools Conjugator is used to change verbs from the infinitive to the past tense.

Because of all of this checking and retrieving, the limericks can take quite a while to load. Animations on the site freeze when the limerick starts loading but fear not for they will unfreeze once the algorithm stops. I am working on making this bit more smooth. And if the limerick does not load, simply try again (or change the name to something more easy-to-rhyme if you are using a custom name) - poetry is hard for machines as well as humans!
